### **Этап 1: Подготовка к работе**

Сначала нужно подготовить рабочее окружение: создать аккаунт на GitHub и установить саму программу Git.

#### **1.1. Регистрация на GitHub**

**GitHub** — сервис для хостинга и совместной работы над IT‑проектами с использованием системы контроля версий Git.

1.  Откройте сайт [github.com](https://github.com/).
2.  Нажмите **"Sign Up"** и пройдите процедуру регистрации. Вам понадобится email, пароль и уникальное имя пользователя (логин).
3.  Подтвердите регистрацию по ссылке в письме, которое придет на вашу почту.

#### **1.2. Установка и настройка Git**

**Git** — это инструмент командной строки, который отслеживает изменения в ваших файлах. Его нужно установить на компьютер.

1.  Зайдите на официальный сайт [git-scm.com](https://git-scm.com/).
2.  Скачайте файл для вашей ОС.
3.  Установите программу. Если вы не уверены в настройках, смело оставляйте все значения по умолчанию — они подходят для большинства задач.
4.  После установки откройте терминал (или **Git Bash** на Windows) и убедитесь, что все работает:
    ```bash
    $ git --version
    git version 2.44.0.windows.1
    ```
5.  **Важнейший шаг:** введите свои данные в системе Git. Это нужно, чтобы все ваши будущие коммиты были подписаны вашим именем.
    ```bash
    $ git config --global user.name "Ваше Имя"
    $ git config --global user.email "ваша_почта@example.com"```

### **Этап 2: Управление проектом на вашем компьютере (Локальный репозиторий)**

Использование Git для ведения проекта на собственном компьютере.

#### **2.1. Основы рабочего процесса**

Это базовый цикл, который вы будете повторять снова и снова.

1.  **`git init`** — Инициализация репозитория.
    Создайте папку для вашего проекта, зайдите в неё через терминал и выполните команду. Это превратит обычную папку в репозиторий Git.
    ```bash
    $ mkdir my-first-project
    $ cd my-first-project
    $ git init
    Initialized empty Git repository in D:/projects/my-first-project/.git/
    ```
2.  **`git status`** — Проверка состояния.
    Эта команда показывает, какие файлы были изменены.
    ```bash
	$ git status
	On branch local
	Changes not staged for commit:
	  (use "git add <file>..." to update what will be committed)
	  (use "git restore <file>..." to discard changes in working directory)
			modified:   Topo2.md

	no changes added to commit (use "git add" and/or "git commit -a")

    ```
3.  **`git add`** — Добавление файлов в "индекс".
    Прежде чем сохранить изменения (сделать "коммит"), вы должны указать Git, какие именно файлы вы хотите сохранить. 
    ```bash
    # Добавить конкретный файл
    $ git add Topo2.md

    # Или добавить все изменённые и новые файлы в папке
    $ git add .
    ```
4.  **`git commit -m "сообщение"`** — Сохранение изменений.
    Это создание "снимка" состояния вашего проекта. Вы фиксируете все файлы, которые добавили через `git add`. Сообщение коммита должно кратко и ясно описывать, что именно вы сделали.
    ```bash
	$ git commit -m "Коммит 1 (master) написание  1 главы"
	[master 1002591] Коммит 1 (master) написание  1 главы
	 1 file changed, 29 insertions(+)

    ```

	#### **2.3. Ветвление (Branching)**

Ветвление — ключевая особенность Git. Оно позволяет работать над новой задачей, не затрагивая стабильную версию кода в основной ветке (`master` или `main`).

*   **`git branch`** — Показывает список всех веток. Звездочка `*` отмечает текущую ветку.
    ```bash
    $ git branch
	  global
	* local
	  master
    ```
*   **`git branch <имя_ветки>`** — Создаёт новую ветку.
    ```bash
    $ git branch new-b
    ```
*   **`git checkout <имя_ветки>`** (или `git switch <имя_ветки>`) — Переключается на указанную ветку.
    ```bash
    $ git checkout -b local
	Switched to a new branch 'local'

    ```
*   **`git checkout -b <имя_ветки>`** — Удобное сокращение: создает новую ветку и сразу переключается на неё.
    ```bash
    $ git switch new-b
    Switched to branch 'new-b'
    ```

*   **`git merge <имя_ветки>`** — Вливает изменения из указанной ветки в текущую.
    *Процесс происходит следующим образом:*
    1.  Вы находитесь в `master`.
    2.  Создаете ветку `new-b`.
    3.  Работаете в ней, делаете коммиты.
    4.  Возвращаетесь в основную ветку (`git switch master`).
    5.  Вливаете всю работу из `new-b` в `master` (`git merge new-b`).

	```bash
	$ git merge global
	Auto-merging Topo2.md
    ```

### **Этап 3: Совместная работа и удалённые репозитории**

Для совместной работы используются удалённые серверы, такие как GitHub.

1.  **Создайте репозиторий на GitHub.**
    *   Зайдите в свой профиль на GitHub и нажмите **"New repository"**.
    *   Дайте ему имя (например, `my-project`), оставьте его публичным и **не ставьте галочку** "Initialize this repository with a README".
    *   GitHub покажет вам URL-адрес вашего нового репозитория (например, `https://github.com/ваш_логин/my-first-project.git`).

2.  **`git remote add origin <URL>`** — Связывание локального и удалённого репозиториев.
    `origin` — это стандартное краткое имя для URL-адреса вашего удалённого репозитория.
    ```bash
    $ git remote add origin https://github.com/ваш_логин/my-project.git
    ```
3.  **`git push -u origin main`** — Отправка ваших коммитов на GitHub.
    Эта команда пушит вашу ветку `main` (или `master`) на удалённый сервер `origin`.
    *   Флаг `-u` устанавливает связь между вашей локальной и удалённой веткой, чтобы в следующий раз вы могли писать просто `git push`.
    ```bash
    $ git push -u origin main
    ...
    To https://github.com/ваш_логин/my-project.git
     * [new branch]      main -> main
    ```
4.  **`git pull`** — Получение изменений с удалённого репозитория.
    Если над проектом работает кто-то ещё и отправил свои изменения на GitHub, эта команда загрузит их и вольёт в вашу локальную версию.

5.  **`git clone <URL>`** — Клонирование существующего репозитория.
    Если вы хотите начать работу над проектом, который уже существует на GitHub, просто скопируйте его URL и выполните эту команду. Она создаст локальную копию проекта на вашем компьютере, уже связанную с удалённым репозиторием.
    ```bash
    $ git clone https://github.com/Ban049/CloneFor2topo
    ```
	
Fork — это операция на сайте GitHub, которая создает полную копию чужого репозитория в вашем личном аккаунте. 
Эта копия принадлежит вам, и вы можете делать с ней всё что угодно, не боясь сломать оригинальный проект.
Необходимо перейти в нужный репозиторий -  нажать fork и создать новый.

Pull Request (PR) — это официальное предложение автору оригинального проекта. 
Вы просите его притянуть изменения из вашего форка и влить их в основной проект.

Как отправить:
1) Открой свой форк на GitHub.
2) Нажми New pull request.
3) Выбери ветку main из оригинального репозитория как место назначения.
4) Добавь название задачи и описание своих правок.
5) Нажми Create pull request.
