### **Этап 3: Совместная работа и удалённые репозитории**

Для совместной работы используются удалённые серверы, такие как GitHub.

1.  **Создайте репозиторий на GitHub.**
    *   Зайдите в свой профиль на GitHub и нажмите **"New repository"**.
    *   Дайте ему имя (например, `my-project`), оставьте его публичным и **не ставьте галочку** "Initialize this repository with a README".
    *   GitHub покажет вам URL-адрес вашего нового репозитория (например, `https://github.com/ваш_логин/my-first-project.git`).

2.  **`git remote add origin <URL>`** — Связывание локального и удалённого репозиториев.
    `origin` — это стандартное краткое имя для URL-адреса вашего удалённого репозитория.
    ```bash
    $ git remote add origin https://github.com/ваш_логин/my-project.git
    ```
3.  **`git push -u origin main`** — Отправка ваших коммитов на GitHub.
    Эта команда пушит вашу ветку `main` (или `master`) на удалённый сервер `origin`.
    *   Флаг `-u` устанавливает связь между вашей локальной и удалённой веткой, чтобы в следующий раз вы могли писать просто `git push`.
    ```bash
    $ git push -u origin main
    ...
    To https://github.com/ваш_логин/my-project.git
     * [new branch]      main -> main
    ```
4.  **`git pull`** — Получение изменений с удалённого репозитория.
    Если над проектом работает кто-то ещё и отправил свои изменения на GitHub, эта команда загрузит их и вольёт в вашу локальную версию.

5.  **`git clone <URL>`** — Клонирование существующего репозитория.
    Если вы хотите начать работу над проектом, который уже существует на GitHub, просто скопируйте его URL и выполните эту команду. Она создаст локальную копию проекта на вашем компьютере, уже связанную с удалённым репозиторием.
    ```bash
    $ git clone https://github.com/Ban049/CloneFor2topo
    ```
	
Fork — это операция на сайте GitHub, которая создает полную копию чужого репозитория в вашем личном аккаунте. 
Эта копия принадлежит вам, и вы можете делать с ней всё что угодно, не боясь сломать оригинальный проект.
Необходимо перейти в нужный репозиторий -  нажать fork и создать новый.

Pull Request (PR) — это официальное предложение автору оригинального проекта. 
Вы просите его притянуть изменения из вашего форка и влить их в основной проект.

Как отправить:
1) Открой свой форк на GitHub.
2) Нажми New pull request.
3) Выбери ветку main из оригинального репозитория как место назначения.
4) Добавь название задачи и описание своих правок.
5) Нажми Create pull request.
